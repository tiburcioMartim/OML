# Introdução ao OML

## Objetivo

O **OML — Orquestrador de Migração Legada** é um framework documental e operacional para conduzir migrações de sistemas legados para stacks modernas com precisão cirúrgica.

## O que o OML é

- Um orquestrador de modernização segura
- Um guardião de regras de negócio
- Um sistema de documentação viva
- Um auditor contínuo de padronização
- Um framework reutilizável e open source

## O que o OML não é

- Um gerador automático de código
- Um script de conversão
- Uma solução mágica que migra tudo sozinha
- Um substituto para decisões humanas

## Princípio fundamental

O legado é a **fonte de verdade funcional**. O novo sistema deve nascer da leitura minuciosa do legado, não da imaginação da IA.

## Como o OML opera

1. Recebe um sistema legado como entrada
2. Investiga telas, banco, permissões, integrações e regras
3. Documenta tudo em dossiês estruturados
4. Propõe modernização preservando comportamentos
5. Só implementa após autorização explícita
6. Revisa, testa e homologa cada parte
7. Continua atuando como guardião após a migração

## Idioma

Todo o OML opera em **português do Brasil**.

Para o perfil do nosso ERP, todo código gerado também deve estar em PT-BR, incluindo classes, funções, variáveis, métodos, constantes, componentes, rotas, comentários, testes, mensagens, documentação e tooltips.
