# /migracao-gerar-relatorio-aprimoramento

## Objetivo

Gerar relatório de melhorias sugeridas para o OML com base na experiência prática de uso.

## Quando usar

Periodicamente (semanal recomendado) ou quando encontrar uma lacuna no agente.

## Entradas esperadas

Experiência acumulada durante a migração.

## Ações obrigatórias

- [ ] Identificar lacunas encontradas no OML
- [ ] Classificar por escopo (núcleo genérico, Laravel, Vue, ERP hospitalar, etc.)
- [ ] Descrever o problema encontrado
- [ ] Sugerir melhoria
- [ ] Avaliar impacto
- [ ] Definir prioridade
- [ ] Indicar se deve virar melhoria no GitHub

## Ações proibidas

- [ ] Não aplicar mudanças no agente matriz sem revisão

## Arquivos que atualiza

- `memoria/relatorio-aprimoramento-agente.md`

"Relatório de aprimoramento gerado com [N] sugestões."
